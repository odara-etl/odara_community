# Feedback, Bugs & Feature Requests

Odara is free and we want it to get better fast. This repository is a public
showcase — **the product source code is private**, so this isn't about
contributing code. It's about telling us what's broken and what you need.

## Found a bug?

[Open a bug report](https://github.com/odara-etl/odara_community/issues/new?template=bug_report.yml).

Before you do, a 30-second checklist that makes a bug fixable:

- **Version** — shown in the app footer / `odara --version`
- **OS** — Windows or Linux, and which version
- **What you did** — the steps, ideally with the node types involved
- **What you expected** vs **what happened**
- **Logs / screenshots** — paste the error text or attach a screenshot

Please don't paste real credentials, connection strings, or production data
into an issue.

## Want a feature?

[Open a feature request](https://github.com/odara-etl/odara_community/issues/new?template=feature_request.yml).

Tell us the **problem** you're trying to solve, not just the solution you have
in mind — it helps us find the best fix. "I need to read from X" is more useful
than "add a button here."

## Security issues

Please **don't** open a public issue for security problems. Email
**info@odara.rs** directly so we can fix it before it's public.
